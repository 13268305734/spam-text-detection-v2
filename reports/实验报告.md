# 基于字符相似性与机器学习的互联网垃圾文本检测系统实验报告

## 摘要

互联网平台中的广告引流、诈骗推广、刷单兼职等垃圾文本会破坏社区生态，并给用户带来安全风险。本项目面向社交评论、论坛短文本等场景，设计并实现一个垃圾文本检测系统。系统采用“规则检测 + 字符相似性变体归一化 + 机器学习分类 + 融合打分”的技术路线，能够识别正常文本与垃圾文本，并对“加薇信”“加 v 信”“领·取优 惠”等规避写法给出风险判断和解释。

本项目提供完整代码、可复现实验脚本、示例数据、Web Demo 和评估结果。当前版本使用课程演示示例数据完成流程验证，正式提交时可替换为真实标注数据重新训练。

## 1. 项目背景与问题定义

在互联网风控场景中，垃圾文本常见于评论区、私信、商品评价和论坛回复。其典型形式包括：

1. 联系方式引流：如“加微信”“加 vx”“私聊”；
2. 利益诱导：如“免费领取”“返现”“红包”“优惠券”；
3. 欺诈推广：如“刷单兼职”“贷款秒到”“博彩开户注册”；
4. 变体规避：如使用同音字、形近字、拼音缩写、插入符号逃避关键词过滤。

本项目将任务定义为二分类任务：

> 输入一条互联网短文本，输出其是否为垃圾文本，并给出风险分数与命中原因。

标签定义如下：

| 标签 | 含义 |
|---|---|
| 0 | 正常文本 |
| 1 | 垃圾文本 |

## 2. 数据集构建与预处理

### 2.1 数据来源

本项目包内提供 `data/sample_comments.csv` 作为可运行示例数据。数据包含正常课程讨论、生活评论、项目交流文本，以及广告引流、优惠返现、兼职刷单等垃圾文本。为了验证变体检测能力，垃圾文本中加入了如下变体增强：

- 字形/同音替换：微信 → 薇信；
- 拼音缩写：微信 → vx / wx；
- 分隔符插入：领取 → 领·取，优惠 → 优 惠；
- 符号扰动：在文本前后加入 emoji 或感叹号；
- 联系方式扰动：URL、手机号、数字 ID。

### 2.2 预处理方法

预处理模块位于 `spam_detector/preprocess.py`，主要步骤包括：

1. Unicode NFKC 标准化，将全角字符转换为半角；
2. 英文字母统一小写；
3. URL、手机号、数字 ID 替换为特殊标记；
4. 去除 HTML 标签；
5. 压缩空白符；
6. 使用字符级建模，保留词内部的细粒度变化。

选择字符级特征的原因是垃圾文本的对抗变体常常发生在词内部，例如“加 薇 信”如果使用普通分词，可能无法稳定切分为有意义的词；而字符级 n-gram 能学习“加/薇/信”等局部组合特征。

## 3. 方法设计

系统总体流程如下：

```text
输入文本
  ↓
文本清洗与归一化
  ↓
规则检测模块
  ↓
字符相似性变体检测模块
  ↓
TF-IDF + Logistic Regression 分类模型
  ↓
融合打分
  ↓
输出：正常/垃圾、风险分数、命中原因
```

### 3.1 规则检测模块

规则模块位于 `spam_detector/rules.py`，检测特征包括：

| 规则类型 | 示例 |
|---|---|
| URL/域名 | `www.example.com`, `.com`, `.cn` |
| 手机号 | `13800138000` |
| 微信/联系方式 | `微信`, `薇信`, `vx`, `wx`, `加v` |
| 利益诱导 | `返现`, `红包`, `优惠券`, `免费领取` |
| 导流行为 | `进群`, `扫码`, `二维码` |
| 高风险词 | `刷单`, `博彩`, `贷款`, `套现` |
| 异常形式 | 重复字符、特殊符号比例过高 |

规则检测优点是可解释性强，能够直接输出命中原因；缺点是泛化能力有限，容易被未见过的新变体绕过。

### 3.2 字符相似性与变体归一化

变体模块位于 `spam_detector/similarity.py`。其核心思想是将疑似规避写法转换为标准写法，再进行风险判断。例如：

| 原文本 | 归一化后 |
|---|---|
| 加 薇 信 | 加微信 |
| 加vx | 加微信 |
| 领·取优 惠 | 领取优惠 |
| V信联系 | 微信联系 |

模块包含：

1. `CONFUSABLE_MAP`：维护字形、同音、拼音缩写映射；
2. `PHRASE_MAP`：维护短语级映射；
3. `normalize_variants()`：执行变体归一化；
4. `variant_score()`：根据归一化后命中关键词、文本变化程度、分隔符数量计算风险分数；
5. `augment_variant()`：用于构造变体增强数据。

该方法虽然不是真正的大规模字符相似性网络，但保留了网络思想的核心：通过字符之间的相似关系连接“异常写法”和“标准写法”，提升对变体垃圾文本的鲁棒性。

### 3.3 机器学习分类模型

机器学习模块位于 `spam_detector/model.py`。本项目使用：

```text
字符级 TF-IDF n-gram + Logistic Regression
```

配置如下：

| 项目 | 设置 |
|---|---|
| 特征 | 字符级 1-4 gram |
| 权重 | TF-IDF + sublinear TF |
| 分类器 | Logistic Regression |
| 类别不平衡处理 | class_weight="balanced" |
| 训练/测试划分 | 75% / 25% |

选择该模型的原因：

1. 训练速度快，适合课程作业复现；
2. 对短文本分类效果稳定；
3. 字符 n-gram 对“加薇信”“加vx”等局部变体具有一定鲁棒性；
4. 可作为 BERT/TextCNN 等深度模型的 baseline。

### 3.4 融合打分

融合模块位于 `spam_detector/ensemble.py`。最终风险分数计算为：

```text
最终风险分数 = 0.30 × 规则分数 + 0.50 × 模型分数 + 0.20 × 变体分数
```

当最终风险分数大于阈值 0.5 时，判定为垃圾文本。

## 6. 结果分析

### 6.1 成功案例

输入：

```text
加薇信免费领取资料，数量有限！
```

系统会将“薇信”归一化为“微信”，并命中“加微信”“免费领取”等关键词，最终判定为垃圾文本。

输入：

```text
私 聊 我 返·现，扫码进群领红包
```

系统能识别分隔符规避行为，并命中“私聊”“返现”“扫码”“进群”等导流风险。

### 6.2 失败案例与原因

可能失败的情况包括：

1. **语义误报**：如“微信小程序开发需要配置环境”包含“微信”，但不一定是垃圾文本；
2. **新型变体漏报**：如果使用未收录的谐音字或表情组合，规则和映射表可能无法覆盖；
3. **长文本上下文不足**：简单关键词难以判断“讨论垃圾广告案例”和“真实垃圾广告”的区别；
4. **示例数据偏简单**：当前数据用于演示流程，真实平台文本更复杂。

### 6.3 优化方向

后续可从以下方向提升：

1. 引入真实标注数据，例如评论区、论坛、短信公开数据；
2. 使用中文 BERT/RoBERTa 进行微调；
3. 构建更完整的字符相似性网络，包括拼音、五笔、OCR 形近、Unicode confusables；
4. 引入上下文语义特征，降低“微信小程序开发”等正常文本误报；
5. 对不同垃圾类型做多分类，如广告、诈骗、引流、低俗等；
6. 设计主动学习流程，让人工复核高不确定样本并迭代模型。

## 7. 创新性说明

本项目的创新点主要体现在：

1. **面向变体垃圾文本**：不仅检测普通关键词，也检测同音/形近/拼音缩写/分隔符规避；
2. **融合式检测框架**：将规则、机器学习、变体归一化三个分数融合；
3. **可解释输出**：不仅给出分类结果，还给出命中原因；
4. **工程化完整**：包含训练、预测、评估、Demo、测试、报告和 Dockerfile；
5. **可拓展**：映射表、关键词表、模型和数据集均可替换或扩展。

## 9. 运行说明

安装依赖：

```bash
pip install -r requirements.txt
```

训练：

```bash
python -m spam_detector.train --data data/sample_comments.csv --model models/tfidf_lr.joblib
```

预测：

```bash
python -m spam_detector.predict --text "加薇信免费领取资料，数量有限！"
```

评估：

```bash
python -m spam_detector.evaluate --data data/sample_comments.csv --model models/tfidf_lr.joblib
```

启动系统：

```bash
streamlit run app.py
```
