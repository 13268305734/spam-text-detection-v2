# 答辩展示提纲：互联网垃圾文本检测系统

## 1. 选题背景

- 互联网评论、论坛、私信中存在大量垃圾文本。
- 常见问题：广告引流、诈骗推广、刷单兼职、优惠返现。
- 难点：垃圾文本会使用变体规避检测，例如“加薇信”“加 v 信”“领·取优 惠”。

## 2. 任务定义

输入一条文本，输出：

1. 是否为垃圾文本；
2. 风险分数；
3. 命中原因；
4. 变体归一化结果。

## 3. 系统架构

```text
文本输入
  ↓
预处理
  ↓
规则检测
  ↓
字符相似性变体归一化
  ↓
TF-IDF + Logistic Regression
  ↓
融合打分
  ↓
结果解释与 Web 展示
```

## 4. 方法一：规则检测

检测 URL、手机号、微信引流、返现优惠、扫码进群、刷单博彩等高风险特征。

## 5. 方法二：字符相似性变体归一化

| 变体 | 归一化 |
|---|---|
| 薇信 | 微信 |
| vx / wx | 微信 |
| 加 v 信 | 加微信 |
| 领·取优 惠 | 领取优惠 |

## 6. 方法三：机器学习分类

- 特征：字符级 TF-IDF 1-4 gram；
- 模型：Logistic Regression；
- 优点：轻量、快速、可复现；
- 后续可升级：TextCNN、BiLSTM、BERT。

## 7. 融合策略

```text
最终风险分数 = 0.30 × 规则分数 + 0.50 × 模型分数 + 0.20 × 变体分数
```

## 9. Demo 展示步骤

1. 输入正常文本：“这个课程讲得挺清楚的，案例也很实用。”
2. 输入普通垃圾文本：“加微信免费领取资料。”
3. 输入变体垃圾文本：“加 薇 信 领·取优 惠。”
4. 展示风险分数、归一化文本和命中原因。

## 11. 不足与展望

不足：

- 示例数据较简单；
- 规则映射表仍需扩展；
- 部分包含“微信”的正常技术文本可能误报。

展望：

- 使用真实标注数据；
- 微调中文 BERT；
- 构建大规模字符相似性网络；
- 做多分类垃圾文本检测。
